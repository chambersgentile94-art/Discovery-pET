# Alcance del MVP - Discovery-pET

## Objetivo del MVP

Construir una primera versión funcional de Discovery-pET que permita validar la idea con usuarios reales en una ciudad o zona piloto.

El MVP debe resolver el flujo principal:

> Un usuario ve, pierde o resguarda una mascota, crea un reporte con foto y ubicación, y otros usuarios pueden visualizarlo en un mapa y colaborar.

## Funciones incluidas

### Usuarios

- Registro de usuario.
- Inicio de sesión.
- Perfil básico.
- Roles iniciales: usuario, voluntario, protectora, administrador.

### Reportes

- Crear reporte de mascota perdida.
- Crear reporte de animal visto.
- Crear reporte de animal abandonado.
- Crear reporte de animal resguardado.
- Crear reporte de animal en adopción.
- Subir fotos.
- Guardar ubicación.
- Editar reportes propios.
- Cerrar reportes propios.

### Mapa

- Mostrar reportes activos en un mapa.
- Abrir ficha desde un marcador.
- Filtrar por categoría, tipo de animal y urgencia.

### Seguimiento

- Cambiar estado del caso.
- Agregar actualizaciones/comentarios.
- Ver historial básico del caso.

### Moderación

- Reportar publicación inválida.
- Ocultar publicación desde rol administrador.
- Marcar caso como publicación inválida.

## Funciones excluidas del MVP

- Inteligencia artificial para comparar imágenes.
- Chat interno avanzado.
- Donaciones.
- QR para collar.
- Panel avanzado para protectoras.
- App web pública.
- Estadísticas municipales.
- Integración directa con WhatsApp Business.

## Criterios de aceptación

El MVP se considera completo cuando:

1. Un usuario puede registrarse e iniciar sesión.
2. Puede crear un reporte con foto y ubicación.
3. El reporte aparece en el mapa.
4. Otro usuario puede abrir la ficha del reporte.
5. Se puede actualizar el estado del caso.
6. Se puede denunciar una publicación.
7. Un administrador puede ocultar publicaciones inválidas.

## Plataforma inicial

La primera plataforma objetivo será Android. iOS queda para una etapa posterior.
